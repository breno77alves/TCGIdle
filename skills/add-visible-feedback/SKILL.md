# Skill: add-visible-feedback

## Purpose
Ensure every meaningful action communicates its state clearly to the player.

## Apply To
- tabs
- buttons
- timers
- expedition start/completion
- deck changes
- duel start/end
- Call Card use
- reroll actions
- disabled states

## Output Format
1. interaction points
2. missing feedback
3. recommended UI states
4. exact labels/CSS behavior
5. validation checklist

## Rules
- every important action should have visible before/during/after states
- errors and lockouts must be explained
- avoid silent successes or failures
