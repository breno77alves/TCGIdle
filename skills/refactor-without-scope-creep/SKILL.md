# Skill: refactor-without-scope-creep

## Purpose
Improve code quality while keeping behavior identical and avoiding accidental feature work.

## Output Format
1. refactor target
2. duplicated/confusing areas
3. exact cleanup changes
4. behavior intentionally preserved
5. verification checklist

## Rules
- do not add features
- do not change state format unless explicitly required
- call out any risk to save/load or UI behavior
