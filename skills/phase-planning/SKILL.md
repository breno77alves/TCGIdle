# Skill: phase-planning

## Purpose
Break a requested task into the smallest product-meaningful implementation slice that fits the current project plan.

## Use When
- planning a new feature
- deciding whether something belongs now or later
- preventing overbuild
- mapping dependencies before coding

## Input
A requested feature or milestone.

## Output Format
1. feature goal
2. fit with current project stage
3. dependencies
4. implementation steps
5. explicitly deferred items
6. manual validation target

## Rules
- align with `PLAN.md`
- do not call anything a prototype
- keep the next slice shippable-feeling and product-facing
- identify persistence impact early
