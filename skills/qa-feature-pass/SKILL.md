# Skill: qa-feature-pass

## Purpose
Run a structured QA pass over a specific feature or milestone.

## Output Format
1. scope tested
2. critical issues
3. major issues
4. minor issues
5. polish opportunities
6. regression risks

## Checklist
- works on fresh save
- works on existing save
- survives reload
- handles empty and populated states
- disabled states behave correctly
- visible feedback is clear
- no obvious console errors

## Rules
- prefer reproducible findings
- separate true bug from polish note
