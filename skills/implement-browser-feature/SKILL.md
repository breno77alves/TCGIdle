# Skill: implement-browser-feature

## Purpose
Implement a feature in plain HTML, CSS, and JavaScript, with no build step, while preserving product polish and localStorage safety.

## Checklist
- respects existing folder structure
- no framework or package tooling
- files affected are explicit
- UI state is visible
- persistence impact is handled
- manual browser testing steps are included

## Output Format
1. files changed
2. implementation notes
3. code or diffs
4. test steps
5. follow-up polish opportunities

## Rules
- prefer small testable changes
- avoid giant rewrites
- do not hide important logic in DOM glue code
- preserve product-facing responsiveness
